lynis分析

1. 简介

Lynis是一个开源的安全工具, 帮助类UNIX系统(Linux, macOS, BSD等)做安全审计(Security auditing), 合规检测(Compliance testing , CIS, PCI, HIPAA, SOx)，漏洞检测(Vulnerability detection)，并提供系统加固(System hardening)的建议。

3. 具体功能

3.1 功能概述

官网上列出了部分: https://cisofy.com/lynis/controls/

具体执行了哪些都在源码目录./include/tests_*中

总共有这些类别: boot_services | kernel | memory_processes | authentication | shells | filesystems | usb | storage | storage_nfs | nameservices | dns | ports_packages | networking | printers_spools | mail_messaging | firewalls | webservers | ssh | snmp | databases | ldap | php | squid | logging | insecure_services | banners | scheduling | accounting | time | crypto | virtualization | containers | mac_frameworks | file_integrity | tooling | malware | file_permissions | homedirs | kernel_hardening | hardening

比如filesystems会做一些文件系统的检测：

FILE-6354 检测/tmp目录下面是否有超过三个月的文件

FILE-6362 检测/tmp目录是否设置了sticky bit

tests_kernel会做一些内核相关的检测：

KRNL-5677 根据/proc/cpuinfo查看是否开启了PAE和NX

KRNL-5723 检测内核加载的模块

tests_malware会检测安装的杀软：

MALW-3275 检测是否安装了 chkrootkit

MALW-3276 检测是否安装了 Rootkit Hunter

tests_logging 会做系统日志相关的检测：

LOGG-2130 检测是否运行了这些syslog相关的守护进程(syslogd|syslog-ng|metalog|systemd-journa)

LOGG-2148 从logrotate的配置文件/etc/logrotate.conf中读取正在处理的日志文件

LOGG-2180 检测lsof开启的日志文件

4.一些分析

4.1 非企业版没有合规检查功能

在配置文件default.prf中:

######################################################################
# Plugins
# ---------------
# Define which plugins are enabled
# - ...
# - Some are for Lynis Enterprise users only
######################################################################


# Lynis plugins to enable
...
plugin=compliance
...
######################################################################
#
# Lynis Enterprise options
# ------------------------
#
######################################################################


# Define which compliance standards are audited and reported on. Disable this if not required.
compliance-standards=cis,hipaa,iso27001,pci-dss

说明这个合规检测功能是企业版才有的。

验证了一下发现：

compliance-standards这个选项在include/profile中读取:

case ${OPTION} in
...
  allow-auto-purge)
  ...
  ;;
  compliance_standards | compliance-standards | check-compliance)  # deprecated: compliance_standards
    COMPLIANCE_STANDARDS_ENABLED=$(echo ${VALUE} | tr ',' ' ')
    or STANDARD in ${COMPLIANCE_STANDARDS_ENABLED}; do
      case ${STANDARD} in
         cis)      COMPLIANCE_ENABLE_CIS=1      ; Debug "Compliance scanning for CIS Benchmarks is enabled" ;;
         ...
       esac
     done
;;

变量COMPLIANCE_ENABLE_CIS只出现在nclude/function文件的ShowComplianceFinding函数中:

ShowComplianceFinding() {
  ...
  case ${STANDARD_NAME} in
            cis)
                if [ ${COMPLIANCE_ENABLE_CIS} -eq 1 ]; then SHOW=1; fi
                STANDARD_FRIENDLY_NAME="CIS"
            ;;  
}

而这个函数并没有被任何模块调用过，可见非企业版确实没有合规审查的功能。
